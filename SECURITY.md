# Security Policy

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

You have two options:

1. **GitHub private vulnerability reporting** (preferred) — use the
   [Report a vulnerability](https://github.com/mahalaxmi-ai/mahalaxmi/security/advisories/new)
   button on the Security tab. Your report goes directly to maintainers and stays private
   until a fix is released.

2. **Email** — send details to **security@mahalaxmi.ai** with the subject line
   `[SECURITY] <brief description>`.

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigations if known

## Response SLA

| Milestone | Target |
|-----------|--------|
| Acknowledgement | 48 hours |
| Status update | 7 days |
| Fix timeline communicated | 14 days |

## Scope

This policy covers the public crates in this repository:
`mahalaxmi-core`, `mahalaxmi-pty`, `mahalaxmi-orchestration`,
`mahalaxmi-detection`, `mahalaxmi-providers`, `mahalaxmi-indexing`,
`mahalaxmi-memory`, `mahalaxmi-cli`, and the VS Code extension.

## Disclosure Policy

ThriveTech follows coordinated disclosure. We ask that you give us reasonable
time to address a vulnerability before any public disclosure. We will credit
researchers who report valid vulnerabilities in our release notes unless they
prefer to remain anonymous.
