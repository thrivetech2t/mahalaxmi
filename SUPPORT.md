# Getting Support

## Before you open an issue

Please check these resources first — your question is likely already answered:

- **[README](README.md)** — quick start, architecture, provider plugin guide
- **[docs/testing.md](docs/testing.md)** — how to run the test suite locally
- **[GitHub Discussions](https://github.com/mahalaxmi-ai/mahalaxmi/discussions)** — community Q&A
- **[Discord](https://discord.gg/bSkzhTPK)** — real-time help from the community and maintainers

## Where to get help

| Type of help | Where to go |
|---|---|
| Usage questions, "how do I…" | [GitHub Discussions](https://github.com/mahalaxmi-ai/mahalaxmi/discussions) or [Discord](https://discord.gg/bSkzhTPK) |
| Bug in a public crate | [GitHub Issues](https://github.com/mahalaxmi-ai/mahalaxmi/issues/new?template=bug_report.md) |
| New provider plugin idea | [Provider Plugin issue](https://github.com/mahalaxmi-ai/mahalaxmi/issues/new?template=provider_plugin.md) |
| Feature request | [GitHub Issues](https://github.com/mahalaxmi-ai/mahalaxmi/issues/new?template=feature_request.md) |
| Security vulnerability | See [SECURITY.md](SECURITY.md) — **do not open a public issue** |
| Corporate CLA / legal | legal@mahalaxmi.ai |
| Everything else | support@mahalaxmi.ai |

## What NOT to file as a GitHub issue

- Questions about the commercial Mahalaxmi product — use support@mahalaxmi.ai
- "It doesn't work" with no reproduction steps — use Discord first to debug
- Security vulnerabilities — use private reporting per [SECURITY.md](SECURITY.md)
